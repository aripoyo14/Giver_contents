# Tech0 Giver Contents Pipeline

Tech0 LINE 登録者向け「ギブ型」コンテンツの収集・ドラフト生成パイプライン。

テック/AI の最新ニュースを初学者向けに噛み砕き、LINE メッセージとして配信するためのコンテンツを半自動で作成する。

## 全体フロー

```
RSS フィード取得（Python / Cron 毎朝8時）
  ↓
inbox/ に記事一覧を Markdown で保存
  ↓
/tech0-giver-draft で5軸スコアリング → 候補10案を drafts/ に出力
  ↓
気になる記事を実際に試す → 結果・メモを tried/ に保存
  ↓
/tech0-giver-tried でチャット入力から LINE コンテンツ1件に仕上げる
  ↓
drafts/ に保存 → GitHub PR でレビュー → LINE へ手動配信
```

`/tech0-giver-draft` はネタ探し・候補出しのためのステップ。LINE 配信するのは実際に試した体験談コンテンツのみ。

## クイックスタート（Claude Code で同じことをやるには）

### 前提

- [Claude Code](https://claude.ai/code) がインストールされていること
- Python 3 と GitHub CLI（`gh`）が使えること

### 手順

**1. リポジトリをクローン**
```bash
git clone <このリポジトリのURL>
cd Giver_contents
```

**2. Python 環境をセットアップ**
```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

**3. 記事を収集する**
```bash
python run_collector.py
# → inbox/YYYY-MM-DD-articles.md が生成される
```

**4. Claude Code を起動してコマンドを実行**
```bash
claude
```

Claude Code のチャットで:
```
/tech0-giver-draft
```
→ inbox の記事がスコアリングされ、候補10案が `drafts/` に保存される。

**5. 気になる記事を実際に試して体験談コンテンツを作る**

試した内容を `tried/YYYY-MM-DD_トピック名/result.md` にメモしたうえで、Claude Code のチャットで:
```
/tech0-giver-tried
```
→ チャットで体験・感想を話しかけるだけで LINE コンテンツ1件が生成され、GitHub PR が作成される。

**6. （任意）Cron で毎朝自動収集**
```bash
bash setup_cron.sh
```

→ 詳細は [docs/SETUP.md](docs/SETUP.md) を参照。

## tried/ フォルダ構成

実際に試した結果（メモ・ログ）を保管する場所。

```
tried/
└── YYYY-MM-DD_トピック名/
    ├── result.md          # 試した手順・感想・惜しかった点
    ├── refinement-log.md  # プロンプト改善の試行ログ（任意）
    └── images/            # 生成した画像・スクリーンショット（git管理外）
        └── *.png
```

> **注:** `images/` 内の画像ファイルは `.gitignore` で除外されているため git には含まれない。

**命名例:**
- `2026-04-15_chatgpt-image-generation/`
- `2026-04-16_claude-cowork-file-organize/`

`result.md` には `/tech0-giver-tried` で使える素材（何をしたか・よかった点・惜しかった点）をメモしておく。

## 記事収集方針

### ソース Tier 定義

| Tier | 役割 | ソース | 重み |
|------|------|--------|------|
| **Tier 1（公式・信頼性重視）** | コンテンツの唯一の情報源。公式一次情報・編集部ありの大手メディアのみ。プロンプトインジェクションリスク回避のためユーザー生成コンテンツは除外 | OpenAI Blog, Google DeepMind Blog, MIT Technology Review, TechCrunch AI, The Verge AI, ITmedia AI+ | 0.8〜1.0 |

### 収集ルール

- **収集期間**: 直近 **14日間**（初学者は毎日ニュースを追わないため、旬の定義を緩めに設定）
- **URL 重複除去**: 同一 URL の記事は自動で排除
- **ソート順**: Tier 昇順（Tier 1 優先）→ 同一 Tier 内は新しい記事順
- **User-Agent**: `Tech0-GiverBot/1.0`、ソース間にディレイを挟んで礼儀正しく取得

## スコアリング方針

### 対象ペルソナ

スコアリングは以下のペルソナを前提に行う。詳細は [docs/user-stories.md](docs/user-stories.md) を参照。

| 項目 | 内容 |
|------|------|
| 属性 | 20〜35歳、社会人または学生 |
| 状況 | Tech0 の説明会・イベント参加済み、LINE 登録済み、受講未決定 |
| 知識レベル | ChatGPT 程度。技術用語は苦手 |
| 心理 | テクノロジーへの関心はあるが「具体的に何をすればいいか分からない」状態 |

### 5軸スコアリング（各 0〜10 点、合計 50 点満点）

| 軸 | 採点基準 | 優先度 |
|----|----------|--------|
| **学習価値** | 「これを知ると何かできるようになる」感があるか | 高 |
| **業務適用度** | 読者が自分の業務課題に当てはめられるか | 高 |
| **試しやすさ** | 読者が明日すぐ試せるか（無料・登録不要・簡単） | 高 |
| **発見価値** | ペルソナにとって初めて知る価値があるか | 中 |
| **初学者親和性** | 技術用語なしで噛み砕いて説明できる内容か | 中 |

合計スコア上位 **10 件** を LINE ドラフトとして採用する。

### 10案の構成バランス

| カテゴリ | 件数 | 役割 |
|----------|------|------|
| すぐ試せる案 | 3〜4件 | 読者が明日無料で試せるツール・機能の紹介 |
| 業務接点が見える案 | 3〜4件 | 「うちでも使えるかも」と思える記事 |
| 視野が広がる案 | 2〜3件 | 社会やキャリアの大きな変化を伝える記事 |

### 除外基準

以下に該当する記事はスコアに関わらず除外する。

- 専門家向けすぎて初学者に届かないもの（論文・実装詳細系）
- 学びに繋がらないゴシップ・炎上系
- 陳腐化した情報（内容が古くなって今さら役立たない）
- **ユーザー生成コンテンツ（Qiita/Zenn 等）**はプロンプトインジェクションリスクのため対象外

### LINE 文面フォーマット

コマンドによってフォーマットが異なる。

**A. 自動生成（`/tech0-giver-draft`）:**
```
【今日のテック/AIニュース】

📌 [トピック名：15文字以内・キャッチー]

[何が起きたか：2〜3行、平易な言葉・技術用語なし]

💬 実際に試してみた——[TODO: ありちゃんが試した感想を追記]

[自分ごと化フレーズ：「〜な人——[具体例]で[できること]。」]
```

**B. 体験談（`/tech0-giver-tried`）:**
```
【今日のテック/AIニュース】

📌 [課題提起：「〜、まだ〇〇してますか？」のような問いかけ]
[読者が「あるある」と感じる痛み・シーン（1〜2行）]
実は今、[解決策の提示（1行）]

💬 実際に試してみた
[体験談：操作→結果→比較→乗り越え方（3〜5行）]

🔑 すぐ使えるコツ N つ（言語化できる場合のみ）
① ...

[締め：「まず〇〇、試してみてください。〜かもしれません。」]
```

**「💬 実際に試してみた」のルール（共通）:**
- 自動生成時は `[TODO: ありちゃんが試した感想を追記]` をプレースホルダーとして入れる
- 良かった点だけでなく、惜しかった点・正直な感想も含める（完璧に見せない）
- 発表のみで未提供の機能は `（まもなく提供予定）` を明記

### トーンガイドライン

- 「平易・前向き・押しつけがましくない」を厳守
- 「〜する人としない人で差がつく」「波に乗るか流されるか」等の煽り・脅し表現は禁止
- 「だから焦れ」ではなく「こうすれば一歩踏み出せる」で締める
- 抽象的な表現（「効率が変わります」「差がつきます」）は禁止。具体的な行動と結果を書く
