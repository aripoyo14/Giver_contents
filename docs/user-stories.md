# ユーザーストーリー：LINE Giver コンテンツ

**対象期間:** LINE 登録済みの状態 → Bootcamp 申込完了まで

---

## ペルソナ

### 基本プロフィール

| 項目 | 内容 |
|------|------|
| 年齢 | 20〜35歳 |
| 属性 | 社会人または学生 |
| 職種 | 特定せず。仕事で成果をあげたい人全般（テクノロジーが今後不可欠になる前提） |
| 知識レベル | ChatGPT は使ったことがある程度。技術用語は苦手 |

### LINE 登録のきっかけ

- Tech0 の説明会やイベントに参加して、その場で登録した
- 知人（Tech0 受講者・関係者）に紹介されて登録した

いずれも「Tech0 に一歩近づいた」タイミングで登録しているが、受講の意思決定には至っていない。

### 現在の状態（2パターン）

| 状態 | 説明 |
|------|------|
| **迷っている（アクティブ）** | Tech0 のことは覚えているが、受講に踏み切れないまま時間が経っている |
| **忘れている（休眠）** | 登録したこと自体をほぼ忘れている。元々は迷っていたが、きっかけがなく離脱した |

両者は本質的に同じペルソナ。休眠ユーザーは「迷った結果、決めきれず放置した人」。

### 受講を迷っている理由

#### 実務的な不安

| 不安 | 具体的な心の声 |
|------|----------------|
| **時間** | 「仕事が忙しくて、学ぶ時間を確保できるか分からない」 |
| **難易度** | 「プログラミングとか自分にできる気がしない。ついていけなかったらどうしよう」 |
| **やりきれるか** | 「以前も自己学習に挑戦して途中で挫折した。また同じことになるのでは」 |

#### アイデンティティ・価値観の迷い

| 不安 | 具体的な心の声 |
|------|----------------|
| **どうしたらいいか分からない** | 「テクノロジーが大事なのは分かっている。でも自分が具体的に何をすればいいのか分からない」 |
| **これが正解か** | 「テクノロジーを学ぶのが本当に正しい選択？他にもっと良い方法があるのでは」 |
| **自分がやる側か** | 「やるべきだとは思うが、自分がやる側の人間なのかが分からない」 |
| **変化への抵抗** | 「学んだら今の仕事のやり方を変えなきゃいけない？今のままでもいいのでは」 |
| **周囲との温度差** | 「周りは誰もやっていない。自分だけ意識高いと思われないか」 |

#### 投資対効果の疑問

| 不安 | 具体的な心の声 |
|------|----------------|
| **実用性** | 「学んだとして、本当に仕事に活かせるのか？」 |
| **長期的な意味** | 「目先の仕事に活きなくても、キャリア全体で見たら意味があるのか？」 |

### 心理状態

- テクノロジーの重要性は感じている。関心もある。しかし「具体的に自分が何をすればいいか分からない」
- 「学ぶべきか迷っている」というより「学びたい気持ちはあるが、一歩目が見えない」
- 明確に「やらない」と決めたわけではない。きっかけさえあれば動く可能性がある
- 解像度が低いまま時間が経ち、迷いが放置されている状態

### このコンテンツで解消したい心理的ハードル

```
「何をすればいいか分からない」→ テック情報を毎朝届け、具体的な一歩のイメージを渡す
「難しそう」       → 平易なコンテンツで「わかる体験」を積み、心理的距離を縮める
「自分がやる側か分からない」 → 専門家ではない人がテクノロジーを活用する実例を伝える
「本当に役立つ？」     → 目先の業務だけでなく、キャリア全体での意味を示す
「やりきれるか不安」    → 1分で読める分量を毎日届け、学びのハードルの低さを体験させる
「変わるのが怖い」     → 変化は一気に起きるものではなく、日々の小さな気づきの積み重ねだと示す
「周りはやっていない」   → 同じ状況から一歩踏み出した人の存在を示唆する
```

---

## ジャーニー全体図

テクノロジーへの「関心はあるが解像度が低い」状態から、コンテンツを通じて解像度が上がっていくプロセス。

```
知る → わかる → 具体的な接点が見える → 自分の業務に当てはめられる
→ もっと知りたくなる → 知るだけでは足りないと感じる → 学びたい → 申込
```

---

## フェーズ 1：知る

**US-01: テックニュースの通知に気づいて開く**
- として: 受講を迷ったまま時間が経っている人 / LINE 登録を忘れていた人
- したい: LINE に届いた通知を何となく開いて、内容を確認したい
- なぜなら: 勧誘ではなく「ちょっと気になる情報」が届いたから
- 感情: 「何だろう？」「あ、このアカウントか」
- 受入条件:
  - [ ] タイトルが 15 文字以内でキャッチーである
  - [ ] 「受講しませんか」等の勧誘文言が含まれていない
  - [ ] 開いた瞬間に読み切れる分量（3〜5行）である

---

## フェーズ 2：わかる

**US-02: 「自分でも理解できた」という感覚を得る**
- として: 技術用語が苦手で、テックニュースは難しいと思い込んでいる人
- したい: テック/AI のニュースが平易な言葉で書かれていて、最後まで読み切りたい
- なぜなら: 「難しくなかった」という体験が、テクノロジーへの心理的距離を縮めるから
- 感情: 「へぇ、そうなんだ」「思ったより難しくなかった」
- 受入条件:
  - [ ] 技術用語・英語略語がそのまま使われていない
  - [ ] 「何が起きたか」が 2〜3 行で説明されている
  - [ ] 1 つのメッセージで 1 つのトピックに絞られている
  - [ ] 読了後の感覚が前向きである（不安を煽らない）

---

## フェーズ 3：具体的な接点が見える

**US-03: テクノロジーと自分の日常が繋がる瞬間に出会う**
- として: テクノロジーに関心はあるが、自分の日常とどう繋がるか具体的に見えていない人
- したい: 職場の雑談や会議でテック/AI の話題が出たとき、内容が分かる自分に気づきたい
- なぜなら: 「知っている」という小さな実感が、漠然としていた関心に具体的な輪郭を与えてくれるから
- 感情: 「あ、これ昨日読んだやつだ」「話についていけた」
- 受入条件:
  - [ ] 取り上げるトピックが社会的に話題性のあるものである
  - [ ] 専門家でなくても会話に参加できるレベルの情報量がある
  - [ ] 「あなたへの影響」が 1 行で具体的に書かれている

---

## フェーズ 4：自分の業務に当てはめられる

**US-04: 読んだ内容が自部署や自分の業務課題と結びつく**
- として: 仕事で成果をあげたいが、テクノロジー活用のイメージが湧かない人
- したい: 記事で紹介されている事例や技術が、自分の業務課題の解決に使えるかもしれないと感じたい
- なぜなら: 「一般的な話」ではなく「自分の現場の話」になった瞬間に、情報の価値が跳ね上がるから
- 感情: 「これ、うちの部署のあの作業に使えるかも」「この課題、AIで解決できるのでは」
- 受入条件:
  - [ ] 特定の職種に限らず幅広い業務に通じる事例が含まれている
  - [ ] 「あなたへの影響」が業務適用をイメージできる粒度で書かれている
  - [ ] 活用シーンが具体的で、読者が自分の現場に置き換えられる

---

**US-05: 紹介されたツールや手法を実際に試してみる**
- として: 記事で知った内容を自分の業務でも使えるかもと感じた人
- したい: 紹介されたツールや機能を実際に触ってみたい
- なぜなら: 「知っている」から「使ってみた」に変わると、テクノロジーが自分のものになる感覚が生まれるから
- 感情: 「ちょっと触ってみよう」「意外と簡単だった」
- 受入条件:
  - [ ] 記事内の事例が、読者が無料で・すぐに試せるものを含んでいる
  - [ ] 試すための心理的ハードルが低い（アカウント登録不要等）
  - [ ] 「使ってみた結果どうなるか」のイメージが記事から読み取れる

---

## フェーズ 5：もっと知りたくなる

**US-06: 毎朝のコンテンツが習慣になり、テックの動向が気になり始める**
- として: フェーズ 1〜4 を数週間繰り返し体験した人
- したい: 朝の流れの中で自然にテック情報をインプットし、次のトピックを楽しみにしたい
- なぜなら: 「わかる→自分ごとになる」体験が積み重なり、テクノロジーへの関心そのものが育ったから
- 感情: 「今日は何の話だろう」「次は何が来るんだろう」
- 受入条件:
  - [ ] 配信が毎日一定の時間帯に届く
  - [ ] 内容のばらつきが少なく、一定の品質が保たれている
  - [ ] 読了までの時間が 1 分以内である

---

## フェーズ 6：知るだけでは足りないと感じる

**US-07: 情報は理解できるが、自分では何も作れない・動かせないと気づく**
- として: テック情報を日常的に読むようになった人
- したい: 「読む側」から「使いこなす側」に近づきたい
- なぜなら: 知識は増えたが、実際に手を動かして何かを作ったり自動化したりする力は自分にはまだないと気づいたから
- 感情: 「知ってるだけじゃ変わらない」「自分でもできるようになりたい」
- 受入条件:
  - [ ] コンテンツの中に「これを使いこなせたら」と想像させる事例がある
  - [ ] 技術を「使う側」の人が特別な存在ではないと感じられる
  - [ ] 「学べばできること」の具体的なイメージが持てる

---

## フェーズ 7：学びたい

**US-08: Bootcamp が「使う側になるための手段」だと認識する**
- として: テクノロジーを自分で使いこなせるようになりたいと感じ始めた人
- したい: 体系的に学ぶ場として Bootcamp の存在を自然に認識したい
- なぜなら: 毎朝のコンテンツで信頼を積み重ねた Tech0 が提供する学びの場なら、自分にも合うかもしれないと思えるから
- 感情: 「Tech0 なら大丈夫かも」「自分にもできるかもしれない」
- 受入条件:
  - [ ] 受講への導線（リンク等）が自然なタイミングで提示される
  - [ ] 「難しそう」ではなく「自分にもできそう」という印象を与える
  - [ ] コンテンツで培った信頼が、学びの場への信頼に自然に接続している

---

## フェーズ 8：申込

**US-09: Bootcamp への申込を完了する**
- として: 受講を前向きに検討しているユーザー
- したい: 申込フォームに迷わずたどり着いて手続きを完了したい
- なぜなら: 情報を「知る→わかる→当てはめる→使いたい」と進んできた延長線上に、学びの場への一歩がある
- 感情: 「やってみよう」
- 受入条件:
  - [ ] 申込フォームへのリンクが明確である
  - [ ] LINE 上での導線がシンプルで迷わない
  - [ ] 申込完了後に「正しい選択をした」と感じられるメッセージがある

## 感情変化サマリー

| フェーズ | 感情の変化 | コンテンツの役割 |
|----------|------------|------------------|
| 知る | 「何だろう？」→ 開いてみる | 勧誘ではない有益情報で接点を作る |
| わかる | 「難しくなかった」→ 読み切れた | 平易な言葉でテクノロジーへの心理的距離を縮める |
| 具体的な接点 | 「これ知ってる」→ 話についていけた | 漠然とした関心に具体的な輪郭を与える |
| 業務に当てはまる | 「うちの業務に使えるかも」→ 自分の現場と結びつく | 一般論から「自分の課題」へ接続する |
| もっと知りたい | 「次は何だろう」→ 毎朝の習慣になる | 継続的に質の高い情報を届け、関心を育てる |

| 学びたい | 「Tech0 なら自分にもできるかも」→ 受講を検討する | コンテンツへの信頼を学びの場への信頼に接続する |
| 申込 | 「やってみよう」→ 申込完了 | 自然な導線で一歩を後押しする |
