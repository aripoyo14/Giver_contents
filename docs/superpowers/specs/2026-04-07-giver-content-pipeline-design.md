**作成日:** 2026-04-07
**プロジェクト:** Tech0 / Marketing / Giverコンテンツ
**ステータス:** 設計確定・実装待ち

---

## 1. 目的・背景

### 目的
テック/AIの最新情報を初学者向けに噛み砕いた「ギブ型」コンテンツを定期配信し、LINE登録者のエンゲージメントを高め、Tech0 Bootcamp受講への転換を促進する。

### 背景
- 現状のLINEはイベント告知・応募が中心で、登録メリットが弱い
- 説明会・イベント経由でLINE登録した約1,000人が休眠状態
- 「有料級の情報」を継続的に届けることで、受講検討を後押しする

### ファネル
```
X投稿（認知）→ LINE登録（関心）→ 毎朝コンテンツ（育成）→ Bootcamp申込（転換）
```

## 2. ペルソナ

**「Tech0に一歩近づいたけど、まだ受講を決めていない人」**

| 項目 | 内容 |
|---|---|
| 属性 | 20〜35歳、社会人または学生 |
| 状況 | Tech0の説明会・イベント参加済み、LINE登録済み、受講未決定 |
| 知識レベル | ChatGPT程度。技術用語は苦手 |
| 心理 | 「学ぶべきか迷っている」「背中を押してほしい」状態 |

**エンゲージメント設計の目標:**
「好きな飲食店の期間限定メニュー・クーポン」のように思わず見てしまうコンテンツ（ファンマーケティング的アプローチ）

## 4. システム構成

### 全体アーキテクチャ

```
[Python スクリプト]         [Claude Code]              [GitHub]
RSS収集・テキスト抽出   →   tech0-giver-draft      →   PR作成
inbox/ に保存               選別・スコアリング           複数人レビュー
（Cron 毎朝8時）            LINE文面生成                 承認・マージ
                            drafts/ に保存               ↓
                                                     LINE手動配信
```

### コンポーネント

| コンポーネント | 役割 | 実装 |
|---|---|---|
| **Collector** | RSS収集・テキスト抽出・inbox保存 | Python + feedparser |
| **tech0-giver-draft** | 選別・生成・drafts保存・PR作成 | Claude Code スキル |
| **GitHub リポジトリ** | レビュー・承認・履歴管理 | GitHub PR フロー |

## 6. 記事選別の軸

| 軸 | 問い | 優先度 |
|---|---|---|
| **学習価値** | 「これを知ると何かできるようになる」感があるか | 高 |
| **生活・仕事への直結度** | 読者の仕事や日常と関係あるか（自分ごと化できるか） | 高 |
| **発見価値** | 読者にとって初めて知る価値があるか | 中 |
| **初学者親和性** | 技術用語なしで噛み砕いて説明できる内容か | 中 |
| **ソース信頼性** | Tier 1か否か | 中 |

**除外基準（NG）:**
- 専門家向けすぎて初学者に届かない（論文・実装詳細系）
- 学びに繋がらないゴシップ・炎上系
- 陳腐化した情報（内容が古くなって今さら役立たない）

## 7. コンテンツ生成設計

### LINE文面フォーマット（テキスト）

```
① 今日のトピック（1行・キャッチー）
② 何が起きた（2〜3行・平易に）
③ あなたの仕事/生活への影響（1行）
```

### プロンプト設計方針

- ペルソナ指定: 「テック/AI初学者、説明会参加済みだが受講未決定の社会人」
- トーン: 平易・前向き・押しつけがましくない
- Tier 2活用: Qiita/Zennの表現を参考にした平易さを指示
- 選別判定: 「初学者に届けるべきか？発見価値スコアを0〜10で返して」

## 8. tech0-giver-draft スキル仕様

### 実行フロー

```
1. inbox/YYYY-MM-DD-articles.md を読む
2. 選別軸に基づいてスコアリング（Claude判断）
3. 上位3件のLINE文面案を生成
4. drafts/YYYY-MM-DD.md に保存
5. GitHubにコミット＆PR作成
```

### 運用モード

| モード | 方法 | 用途 |
|---|---|---|
| **インタラクティブ** | Claude Codeで `/tech0-giver-draft` 実行 | 文面を会話で調整、品質確認期 |
| **ヘッドレス（自動）** | `claude --print` でCronから起動 | 安定後の自動化 |

**移行方針:** プロトタイプはインタラクティブから開始 → 品質安定後にCron自動化
