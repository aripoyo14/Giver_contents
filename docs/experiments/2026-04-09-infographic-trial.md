# 実験レポート：LINEニュース向けインフォグラフィック自動生成

**実施日:** 2026-04-09
**ステータス:** ペンディング

---

## 背景・目的

2〜3分で読めるLINEニュースに視覚情報を加えることで視認性が上がるかを検証する。
ユーザーストーリー上のペルソナ（技術用語が苦手・1分以内に読み切りたい）に対して、インフォグラフィックが有効かどうかを実際に作って判断する。

---

## やったこと

- Python（matplotlib + Pillow）でLINE向けPNG画像を自動生成するスクリプトを試作
- 題材: `drafts/2026-04-09.md` 案1「ChatGPT × 日常アプリ連携」
- 出力: `infographics/2026-04-09-chatgpt-apps.png`
- スクリプト: `infographics/gen_chatgpt_apps.py`

---

## 結果


| 観点      | 結果                               |
| ------- | -------------------------------- |
| 技術的可否   | 生成自体は可能。PNG出力まで動く                |
| 絵文字     | matplotlibでは描画不可（文字化け）           |
| 日本語フォント | Hiragino Sansを明示指定すれば表示できる       |
| 品質      | ブランドロゴが使えないため「何のアプリか伝わらない」カードになる |
| 運用コスト   | 記事ごとにスクリプトを調整する手間がかかる            |


---

## 考察

### ユーザーストーリー視点

- **フェーズ1（知る）**: 画像付きメッセージは通知プレビューで目を引く可能性あり
- **フェーズ2（わかる）**: 数値・比較の図解は「難しくなかった」体験を補強できる
- ただし、ペルソナの最大ハードル「難しそう」はすでに平易な文章で対処できており、インフォグラフィックの追加効果は限定的

### 品質面

- アプリ紹介・サービス紹介系はブランドロゴが必須になるが著作権の問題がある
- matplotlibでのレイアウト調整がピクセルレベルで煩雑で、毎日の運用には向かない

---

## 結論

**現時点でLINE配信品質に乗せるには追加工数が大きく、費用対効果は低い。**

### 用途別の向き・不向き


| 用途               | 判定     | 理由               |
| ---------------- | ------ | ---------------- |
| 数値グラフ系（AI市場規模など） | 向いている  | データをそのまま可視化できる   |
| アプリ紹介・サービス比較     | 向いていない | ロゴ・アイコンが使えず伝わらない |
| 毎日の自動生成          | 向いていない | 記事ごとの調整コストが高い    |
