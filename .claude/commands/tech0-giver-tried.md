# tech0-giver-tried

ありちゃんが「実際に試してみた」体験・感想をチャットで受け取り、LINE Giverコンテンツ1件を生成する。

## このコマンドの使い方

ありちゃんがチャットで以下を話しかけるだけでOK:
- 試したツール・機能の名前（または記事URL）
- 実際にやってみたこと
- 良かった点・惜しかった点
- あれば「もっとこうなればいいのに」という一言

例:
> 「Notionの議事録自動生成を試してみた。会議中に録音しておくと終わったあとに要約してくれる。日本語の精度が思ったより良かった。ただ専門用語は聞き取り間違える。Qiitaで紹介されてた。」

## 実行手順

### Step 0: ユーザーストーリーを確認する

`docs/user-stories.md` を読み、ペルソナ（関心はあるが「何をすればいいか分からない」状態）を頭に入れる。

### Step 1: ありちゃんの入力を整理する

チャット内容から以下を読み取る:

| 項目 | 内容 |
|------|------|
| トピック | 何のツール・機能・話題か |
| 元記事 | URL またはタイトル（ない場合は「ありちゃん体験談」として扱う） |
| 試した目的・背景 | なぜ試したか（「〇〇のために」という文脈） |
| やってみたこと | 実際に試した操作・手順 |
| 良かった点 | うまくいったこと・驚いたこと |
| 惜しかった点 | 不満・限界・改善してほしい点 |
| 比較対象 | 他のツールや以前の方法との違い（あれば） |
| 発表のみか | 実際に使えるか、発表・予告だけか |

### Step 2: LINE文面を生成する

**フォーマット（厳守）:**
```
【今日のテック/AIニュース】

📌 [課題提起：「〜、まだ〇〇してますか？」「〜、こんな経験ありませんか？」のような問いかけ]
[読者が「あるある」と感じる具体的な痛み・シーン（1〜2行）]
実は今、[解決策の提示（1行）]

💬 実際に試してみた
[体験談：具体的な操作・結果・比較・正直な感想（3〜5行、話し言葉）]

🔑 すぐ使えるコツ N つ（※コツが言語化できる場合のみ）
① ...
② ...
③ ...

[締め：「まず〇〇、試してみてください。〜かもしれません。」]
```

**📌 タイトル（課題提起）の書き方ルール:**
- 読者の課題・痛みを問いかける形にする（「〜、まだ〜してますか？」「〜で困っていませんか？」）
- タイトル単体でトピックを言い切らず、「この先を読みたい」と思わせる引きを作る
- タイトルの直後に、読者が「あるある」と感じる具体的なシーンや痛みを1〜2行で描写する
- 「実は今、〜」で解決策を提示し、問題→解決の流れを作る

**「💬 実際に試してみた」の書き方ルール:**
- 「——」ダッシュは使わず、次の行から体験談を書く
- ありちゃんの言葉をもとに、一人称（「試してみたら」「入力してみると」）で整える
- 体験談は3〜5行を目安に。「操作→結果→ツール比較→どう乗り越えたか」の流れが理想
- 良かった点だけでなく、惜しかった点・正直な感想も含める（完璧に見せない）
- 「すごかった」「便利だった」の一言で終わらず、具体的な場面・描写・比較を入れる（例：「白黒の棒人間が出てきて全然ダメ……」）
- 複数ツールを比較した場合はその対比を活かす（ツールA→だめだった、ツールB→良かった、という流れ）
- 一発で完成するような誇張はしない（「数ターン必要だった」という正直さが信頼につながる）
- 文末は短文・話しかけるトーンで（「〜でした」「〜でしたよ」。「〜になりました」のような生成AIっぽい丁寧語は避ける）
- 「！」「……」を自然に使い、感情の起伏を表現する

**「🔑 すぐ使えるコツ」の書き方ルール:**
- 体験から得た再現性のある学びが言語化できる場合のみ追加（任意）
- ① ② ③ の箇条書きで3つ以内
- 「場面を具体的に書く」「AIにプロンプト自体を改善してもらう」のように、すぐ行動できる粒度で書く

**締めの書き方ルール:**
- 「まず〇〇、試してみてください。〜かもしれません。」形式で行動を促す
- 自分ごと化フレーズは独立させず、締めの一文に自然に統合する
- 「仕事の効率が変わります」「差がつきます」のような抽象表現は禁止

**事実確認ルール:**
- 「発表のみ」で実際に使えない場合は `（現在準備中、まもなく提供予定）` を解決策提示部分に明記
- ありちゃんが実際に試していない機能については「試してみた」と書かない

**文体ガイドライン:**
- ありちゃんのnote記事の文体に合わせる：短文の連続・「！」「……」・話しかけるトーン
- 生成AIっぽい丁寧語（「〜になりました」「〜されています」「〜できます」の多用）は避ける
- 期待感を持たせる締めが有効（「〜かもしれません」「〜になるかも」）

**トーン:**
- 煽り・脅し表現は禁止（「波に乗るか流されるか」等）
- 読後感は「へぇ、面白い」「ちょっと試してみようかな」

### Step 3: drafts/ に保存する

`drafts/YYYY-MM-DD.md` に追記する（同日既存ファイルがあれば末尾に追記、なければ新規作成）。

```markdown
# LINE Giverコンテンツ案 YYYY-MM-DD

> 生成日時: YYYY-MM-DD HH:MM
> 種別: ありちゃん体験談

---

## 案（カテゴリ: すぐ試せる｜元記事: [タイトルまたはURL]）

[LINE文面]

---
```

### Step 4: GitHubにコミットしてPRを作成する

```bash
git add drafts/YYYY-MM-DD.md
git commit -m "draft: YYYY-MM-DD ありちゃん体験談コンテンツ案"
gh pr create \
  --title "[DRAFT] YYYY-MM-DD Giver LINE ありちゃん体験談" \
  --body "## 概要
ありちゃんが実際に試したツール・機能をもとに生成したコンテンツ案です。

## レビュー依頼
- [ ] 「💬 実際に試してみた」の文面・トーンの確認
- [ ] 事実誤認がないか（発表のみのものを「使える」と書いていないか）
- [ ] 自分ごと化フレーズの具体性確認
- [ ] 承認後、LINEへ手動配信"
```
