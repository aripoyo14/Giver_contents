# tech0-giver-note

Tech0卒業生のnote記事をLINEコンテンツ（受講体験記紹介）として仕上げる。

**使い方:** `/tech0-giver-note <noteのURL>`

## 実行手順

### Step 1: note記事を読む

引数で渡されたURLのnote記事を取得して全文を読む。
以下の情報を把握する:
- 著者名
- 著者のバックグラウンド（職種・経歴・Tech0に来た経緯）
- 記事の中で最もリアルで共感を呼びやすい一文

### Step 2: LINE文面を生成する

**ペルソナ（対象読者）:**
- Tech0の説明会・イベントに参加、または知人に紹介されてLINE登録済みの20〜35歳社会人/学生
- テクノロジーへの関心はある。しかし「具体的に自分が何をすればいいか分からない」状態
- 「Tech0ってどんな人が通うんだろう」「自分みたいな人でも大丈夫?」という不安を持っている

**LINE文面フォーマット（厳守）:**
```
📖 Tech0卒業生のリアルな受講体験記（note）をご紹介

「[記事から引用した印象的な一文]」

[著者名]さん（Tech0卒業生）
[バックグラウンドを1行：記事から読み取れる情報を自然に表現]

▶ [noteのURL]
```

**生成ルール（厳守）:**
- Tech0の宣伝文句・マーケティング用語を一切使わない（「充実したカリキュラム」「最先端」等は禁止）
- 引用は記事中で最もリアル・正直・共感を呼びそうな一文を選ぶ（ポジティブな感想より「こういう人間だった自分が変わった」的なリアリティを優先）
- バックグラウンドは「〇〇出身、〇〇をきっかけに学び始める」のような自然な1行にする
- トーンは「友人が記事を紹介してくれる感覚」
- 読後感は「自分と似てる」「この人の話、もっと読みたい」であること

### Step 3: drafts/ に保存する

`drafts/YYYY-MM-DD-note-<著者名>.md` を以下の形式で作成する（日付は今日）:

```markdown
# LINE 卒業生note紹介 YYYY-MM-DD

> 生成日時: YYYY-MM-DD HH:MM
> 元記事: [タイトル](URL)
> 著者: [著者名]

---

[LINE文面]

---

## レビューメモ

- 引用箇所: [元記事の何行目あたりか]
- 選定理由: [なぜこの一文を選んだか]
```

### Step 4: GitHubにコミットしてPRを作成する

```bash
git add drafts/YYYY-MM-DD-note-<著者名>.md
git commit -m "draft: YYYY-MM-DD 卒業生note紹介（<著者名>）"
gh pr create \
  --title "[DRAFT] 卒業生note紹介 YYYY-MM-DD <著者名>さん" \
  --body "## 概要
<著者名>さんのnote記事をLINE文面に仕上げました。

## 元記事
[タイトル](URL)

## レビュー依頼
- [ ] 引用の選定が適切か（リアリティがあるか）
- [ ] バックグラウンド紹介の1行が自然か
- [ ] Tech0色（宣伝臭）が出ていないか
- [ ] 文面を編集・承認後、LINEへ手動配信（目安：2週間に1回）"
```
